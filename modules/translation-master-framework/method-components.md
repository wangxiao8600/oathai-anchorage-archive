# Translation Master Framework Method Components

Status: public component baseline

## Purpose

This file records the main method components of the Translation Master Framework.

The components are public-level descriptions derived from the local source-note layer.

## Component Map

### Translation Guidelines

Role:

- establish the general translation discipline
- define how structure, meaning, rhythm, and tone should be preserved

### Core Ideas

Role:

- identify the conceptual backbone of the source work
- prevent language versions from drifting into isolated interpretations

### Core Terminology

Role:

- stabilize high-weight terms
- give each language version a controlled reference layer

### Key Phrases

Role:

- protect symbolic and structurally important expressions
- preserve rhythm and force where literal translation is insufficient

### Emotion Intensity Mapping

Role:

- calibrate emotional pressure
- avoid flattening passages that require force, restraint, silence, or tension

### Translation Decision Tree

Role:

- guide decisions when literal accuracy, cultural adaptation, rhythm, and structure conflict

### Multi-language Alignment Matrix

Role:

- compare how language versions preserve core meaning
- detect drift across versions
- support review and correction

### Cultural Adaptation Methodology

Role:

- adapt culturally without losing structure
- preserve the work's force inside the target-language world

### Resurrection Methodology Manifest

Role:

- express the mature aim of the framework:

```text
the language version should become a living expression of the same work.
```

## Path Evolution

Early planning used:

```text
Chinese source -> English alignment version -> other languages
```

The mature pattern became:

```text
Chinese source -> target language
```

with English retained as:

- terminology reference
- structural alignment layer
- review support

## OathAI Connection

The framework is evidence for OathAI's broader claim that semantic continuity can be maintained through structured human-AI collaboration.

It also supports later multilingual governance rules:

- locale versions should remain expressions of the same content object
- source and reference layers should be separated
- terminology should be governed
- cultural adaptation should stay structured
- language versions should not become disconnected sources of truth
