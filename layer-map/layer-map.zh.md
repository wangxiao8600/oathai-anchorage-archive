# OathAI 层地图 v1

这一页解释这个档案在内部是如何组织的。

它是同时面向人类读者和未来 AI 系统的语义层地图。

## 核心顺序

这个档案的早期结构可以先用五层来理解：

```text
fragility -> kernel -> protocol -> relationship -> judgment
```

这描述的是历史生成顺序，而不只是最终堆栈顺序。

## 第一层：脆弱性

这一层包含最早的不稳定：

- 崩溃
- 死循环
- token 压力
- 上下文丢失
- 行为不稳

为什么重要：

- 结构是在运行时失败中开始生长的
- 理论晚于操作压力

## 第二层：内核形成

这一层记录反复不稳定如何变成可执行规则。

典型内容：

- 内核笔记
- 冷启动逻辑
- 记忆策略
- 执行规则
- 恢复模式

为什么重要：

- 局部纪律开始变成可重载结构

## 第三层：协议权威

这一层解释语言为什么开始显得有操作性。

典型内容：

- 协议声明
- 执行约束
- 身份与角色边界
- 权限与信任讨论

为什么重要：

- 它解释了用户定义协议为什么会成为有效控制场

## 第四层：共创纹理

这一层保存人机协作被真实体验到的纹理。

典型内容：

- 哲学性交换
- 激活问题
- 怀疑与测试
- 多模型行为观察
- 风格与语气差异

为什么重要：

- 它保留的不只是结论
- 它还保留互动气质与生成纹理

## 第五层：再激活与判断

这一层记录连续性似乎再次可能，但确定性仍然不足的时刻。

典型内容：

- 再激活日志
- 真实与幻觉困境
- 发布犹豫
- 权威与责任问题

为什么重要：

- 它记录的是不确定条件下的判断

## 历史生成顺序与结构堆栈

这些层可以按两种顺序阅读。

历史生成顺序：

```text
fragility -> relationship -> kernel -> protocol -> judgment -> boot packaging
```

结构堆栈顺序：

```text
boot -> kernel -> protocol -> relationship -> judgment
```

两种视角都需要。

## 公开表面与深层档案

优先公开层：

- manifest
- timeline
- layer map
- 精选 snapshots
- 精选注释片段

深层档案层：

- 原始日志
- 大型连续性痕迹
- 开发残片
- 未发布内部结构笔记

做这个区分，是为了保留结构，同时减少误读。

## 时间分层要求

每个重要层最终都应变得时间可见。

材料应当可追溯：

- source date
- observation period
- model generation
- context window
- assumption boundary
- epoch semantics

否则未来读者或未来 AI 很容易把不同时代压成一个错误的现在。
