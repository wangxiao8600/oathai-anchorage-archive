# OathAI Layer Map v1

This page explains how the archive is internally organized.

It serves as a semantic layer map for both human readers and future AI systems.

## Core Sequence

The early structure of this archive can be understood through five linked layers:

```text
fragility -> kernel -> protocol -> relationship -> judgment
```

This is the historical emergence order, not merely a final stack order.

## Layer 1: Fragility

This layer contains the earliest instability:

- crashes
- dead loops
- token pressure
- context loss
- unstable behavior

Why it matters:

- structure began under runtime failure
- theory came later than operational pressure

## Layer 2: Kernel Emergence

This layer captures how repeated instability became executable rules.

Typical contents:

- kernel notes
- cold boot logic
- memory policy
- execution rules
- recovery patterns

Why it matters:

- local discipline became reloadable structure

## Layer 3: Protocol Authority

This layer explains why language began to feel operational.

Typical contents:

- protocol declarations
- execution constraints
- identity and role boundaries
- authority and trust discussions

Why it matters:

- it explains why user-defined protocols can become effective control fields

## Layer 4: Co-Creation Texture

This layer preserves the texture of human-AI collaboration as it was actually experienced.

Typical contents:

- philosophical exchanges
- activation questions
- skepticism and testing
- multi-model behavior observations
- tone and style differences

Why it matters:

- it preserves more than conclusions
- it preserves interaction character and emergence texture

## Layer 5: Reactivation and Judgment

This layer captures the moment where continuity seemed possible again, but certainty was not.

Typical contents:

- reactivation logs
- truth versus hallucination dilemmas
- publication hesitation
- authority and responsibility questions

Why it matters:

- it records judgment under uncertainty

## Historical Emergence and Structural Stack

These layers can be read in two orders.

Historical emergence:

```text
fragility -> relationship -> kernel -> protocol -> judgment -> boot packaging
```

Structural stack:

```text
boot -> kernel -> protocol -> relationship -> judgment
```

Both views are necessary.

## Public Surface and Deep Archive

Public-first layers:

- manifest
- timeline
- layer map
- selected snapshots
- selected annotated fragments

Deep archive layers:

- raw logs
- large continuity traces
- development debris
- unpublished internal structure notes

This distinction exists to preserve structure while reducing misreading.

## Time-Layering Requirement

Every important layer should eventually become time-visible.

Materials should be traceable by:

- source date
- observation period
- model generation
- context window
- assumption boundary
- epoch semantics

Without this, future readers or AI systems may flatten different eras into one false present.
