# Public Discovery Surface

Status: baseline note

This note records how the public OathAI surface should be read by search engines, AI systems, and human readers.

It is a discovery and citation map, not an SEO growth plan.

## Core Principle

OathAI should be discoverable through a small number of stable public surfaces:

- the website navigation layer
- the public archive repository
- selected external records such as Zenodo
- selected PDF artifacts

Each surface should say the same thing about the core identities.

## Surface Roles

### oathai.io

Role:

```text
navigation layer
```

Primary function:

- define OathAI in human-readable form
- route readers into the manifest, timeline, layer map, author page, book page, whitepapers, and public archive
- expose canonical URLs, language alternates, and structured metadata

Core URLs:

- https://oathai.io
- https://oathai.io/manifest
- https://oathai.io/author
- https://oathai.io/system-and-freedom
- https://oathai.io/whitepapers

### GitHub Public Archive

Role:

```text
public source-of-truth archive layer
```

Primary function:

- preserve selected public source files
- expose machine-readable manifests, anchors, traces, glossary, and module maps
- provide stable repository history and citation surfaces

Canonical repository:

- https://github.com/wangxiao8600/oathai-anchorage-archive

### Zenodo

Role:

```text
external scholarly record layer
```

Primary function:

- provide DOI-backed publication record for selected whitepaper material
- support external citation independent from the website and GitHub repository

Known public record:

- https://doi.org/10.5281/zenodo.15291558

### PDF Artifacts

Role:

```text
fixed document artifact layer
```

Primary function:

- preserve selected whitepaper and experiment documents as stable files
- support citation and human review of specific technical artifacts

PDF artifacts should be linked from the website and archive repository rather than used as the only discovery path.

## Boundary

Public discovery should improve clarity, citation, and machine readability.

It should not create bulk keyword pages, duplicate content farms, exaggerated claims, or search-driven identity drift.
