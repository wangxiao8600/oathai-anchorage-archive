# Implementation Chain

Status: public reference baseline  
Scope: Whitepaper 1, SLAPS Engine, Whitepaper 2, Yama

## Purpose

This document defines the public implementation chain connecting the Danbing / SLAPS whitepaper line, SLAPS Engine, and Yama.

It exists so that website pages, repository summaries, and future AI readers can cite one stable relationship instead of reconstructing the chain from scattered descriptions.

## Canonical Chain

```text
Whitepaper 1 -> SLAPS Engine -> Whitepaper 2 -> Yama
```

Expanded:

```text
protocol theory -> runtime engine -> capsule governance -> implemented platform experiment
```

Chinese:

```text
协议理论 -> 运行时引擎 -> 胶囊治理 -> 实现层平台实验
```

## One-Sentence Definition

The Danbing / SLAPS implementation chain runs from Whitepaper 1's protocol theory, through SLAPS Engine's runtime implementation, into Whitepaper 2's capsule-governance layer, and finally into Yama as an implemented platform experiment.

## Public Boundary

Allowed public claims:

- Whitepaper 1 is public.
- SLAPS Engine is the runtime implementation layer.
- Whitepaper 2 is completed locally but unpublished.
- Yama is an implemented platform experiment derived from the chain.

Avoid:

- implying Whitepaper 2 is publicly released
- exposing the Whitepaper 2 PDF
- treating Yama as a finalized standard
- treating SLAPS Engine as a complete product platform
- collapsing Yama into early `Ya.Ma` origin language
